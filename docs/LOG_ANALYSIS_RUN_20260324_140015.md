### 2.4 GPS 约束

#### 2.4.1 数据源

- **配置**: `sensor.gps.enabled=true`，`topic=/ublox/fix`
- **Bag**: rosbag2_player 因 `ublox_msgs` 未安装而忽略 `/ublox/navpvt` 等，但 `/ublox/fix` 使用 `sensor_msgs/NavSatFix`，未被忽略
- **首条 GPS**: `[LivoBridge][GPS] First valid GPS received: ts=1628249881.000 lat=31.026059 lon=121.436794`
- **结论**: GPS 数据流入正常。

#### 2.4.2 关键帧-GPS 关联失败（根因）

- **现象**: 全部关键帧 `has_gps=0`
- **日志**:
  - `processKeyframeCreate kf_id=0..299 gps_aligned=1 has_gps=0`
  - `[HBA][GTSAM][GPS_FILTER] reject_no_valid=300 added=0`
- **根因**: `FrontEndModule::gpsCacheGet()` 使用硬编码 `max_dt=0.5s`，而配置 `gps.keyframe_match_window_s=2.5s`。0.5s 过严，导致关键帧时间戳与 GPS 样本匹配失败。
- **修复**: 已改为使用 `ConfigManager::instance().gpsKeyframeMatchWindowS()`，与 GPSManager 一致。

#### 2.4.3 HBA GPS 杠杆臂

- **配置**: `gps.lever_arm_imu=[0.1743, 0.0053, 0.9063]`，ConfigManager 读取正确
- **日志**: `[HBA] GPS lever arm is zero`
- **可能原因**: HBA `init()` 时 config 未就绪，或离线补丁 YAML 结构差异；`runGTSAMFallback` 内已再次调用 `resolveGpsLeverArmForHba`。当 `lever_arm=0` 时，GPS 因子仍可添加（pos_map=pos_enu），主要瓶颈在 `has_valid_gps`。
