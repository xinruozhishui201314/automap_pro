## 1. 致命问题：ConfigManager GPS 配置加载异常

### 现象（full.log L148–151）

```
[ConfigManager][GPS_CONFIG_DUMP] Failed to load GPS config: yaml-cpp: error at line 238, column 27: operator[] call on a scalar (key: "quality_threshold_hdop")
[ConfigManager][EXCEPTION] YAML exception loading '...system_config_M2DGR.yaml': ... operator[] call on a scalar (key: "quality_threshold_hdop")
```

### 原因

- 在 `config_manager.cpp` 的 GPS 配置块中，先用 `get("gps.xxx")` 读入缓存，再用校验 lambda 对 `gps_node[yaml_key]` 做直接访问以核对缓存与 YAML 一致。
- yaml-cpp 在以下情况可能使 `cfg_["gps"]` 或其后对 `gps_node` 的 `operator[]` 遇到“对 scalar 使用 operator[]”的语义（例如重复键、解析顺序、或实现细节），从而在 `gps_node["quality_threshold_hdop"]` 处抛错。

### 修复（已做）

- **文件**：`automap_pro/src/core/config_manager.cpp`
- **改动**：校验逻辑不再使用 `gps_node[yaml_key]`，改为用路径 `get<double>("gps." + yaml_key, cached)` 读回，再与缓存比较。这样不再对可能为 scalar 的节点做 `operator[]`，避免该异常。

### 验证建议

- 重新运行同场景（同一 config、同一 bag），确认不再出现上述 YAML 异常，且 `[ConfigManager][GPS_CONFIG_DUMP]` 正常打印并完成 load。

## 4. 风险与回滚

- **风险**：校验改为仅依赖 `get()` 路径读回，若 YAML 中 `gps` 段存在重复键或异常结构，`get()` 可能回退到默认值，校验会以“缓存与 get() 一致”通过，不会因 operator[] 再抛错。
- **回滚**：若需恢复“直接对 gps_node 下标校验”，可还原对 `gps_node[yaml_key]` 的调用，并同时排查 YAML 中是否存在重复 `gps` 键或缩进错误。
