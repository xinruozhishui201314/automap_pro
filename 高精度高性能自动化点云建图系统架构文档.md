## 1. 系统概述

### 1.1 系统目标

构建一个面向城市道路、园区、隧道、矿山等复杂场景的**自动化高精度三维点云建图系统**，满足以下核心需求：

- **高精度**：全局一致性误差 < 0.3%（相对于轨迹长度）
- **高鲁棒**：适应 GPS 信号时好时坏、退化场景（长走廊、隧道）
- **高性能**：前端实时 ≥ 10Hz，全流程可在线运行
- **增量式**：支持多次采集数据的增量式拼接建图
- **自动化**：端到端无人工干预完成建图

### 1.2 核心模块一览

| 模块                   | 技术选型                | 角色                   |
| ---------------------- | ----------------------- | ---------------------- |
| 前端里程计             | **Fast-LIVO2**          | LiDAR-IMU-(Visual) 紧耦合状态估计 |
| GPS 融合               | 自适应因子图融合         | 弱/强 GPS 信号自适应融合 |
| 增量式子地图管理       | **MS-Mapping**          | 多会话增量建图、子地图管理 |
| 回环粗匹配             | **OverlapTransformer**  | 基于深度学习的位置识别 |
| 回环精匹配             | **TEASER++**            | 鲁棒点云配准           |
| 全局一致性优化         | **HBA**                 | 分层位姿图优化 + BA    |
| 地图输出               | 自研                    | 多格式高精度地图输出   |

### 1.3 输入与输出

#### 输入

| 数据源   | 频率         | 必选/可选 | 说明                              |
| -------- | ------------ | --------- | --------------------------------- |
| LiDAR    | 10–20 Hz     | **必选**  | 主几何信息源                       |
| IMU      | 200–400 Hz   | **必选**  | 高频姿态传播                       |
| GPS/GNSS | 1–10 Hz      | **必选**  | 全局坐标约束（信号质量不稳定）     |
| Camera   | 20–30 Hz     | 可选      | 视觉辅助约束（Fast-LIVO2 支持）   |

#### 输出

| 输出项           | 格式                | 说明                     |
| ---------------- | ------------------- | ------------------------ |
| 全局点云地图     | PCD / PLY / LAS     | 高精度三维点云           |
| 优化后轨迹       | TUM / KITTI 格式     | 带时间戳的6DoF位姿       |
| 子地图集合       | 分块PCD + 位姿JSON  | 支持增量更新             |
| 位姿图           | g2o / JSON           | 可视化与二次优化         |
| 回环检测报告     | JSON / CSV           | 回环对及置信度           |

3. 模块详细设计
3.1 传感器接入与预处理模块
3.1.1 功能职责
多传感器数据接收与缓存
硬件/软件时间同步
LiDAR 运动畸变补偿
GPS 坐标系转换与质量评估
数据有效性校验
3.1.2 时间同步策略
软件时间对齐
  • 以 LiDAR 时间戳为基准
  • IMU: 线性插值对齐
  • Camera: 最近邻匹配
  • GPS: 最近邻匹配 + 延迟补偿
3.1.3 LiDAR 运动补偿
输入: 原始点云 P_raw, IMU 数据队列
处理:
  1. 取当前帧时间范围 [t_start, t_end]
  2. IMU 积分得到帧内相对运动
  3. 对每个点按时间戳插值补偿
输出: 去畸变点云 P_undistorted

3.1.4 GPS 坐标转换
WGS84 (lat, lon, alt)
    │
    ▼ GeographicLib / proj4
ENU (East, North, Up)
    │
    ▼ 初始化原点对齐
Local Frame (x, y, z) — 与里程计坐标系对齐

3.1.5 GPS 质量评估
enum class GPSQuality {
    INVALID    = 0,   // 无效信号
    LOW        = 1,   // 单点定位, HDOP > 5.0
    MEDIUM     = 2,   // DGPS, 2.0 < HDOP ≤ 5.0
    HIGH       = 3,   // RTK Float, 1.0 < HDOP ≤ 2.0
    EXCELLENT  = 4    // RTK Fixed, HDOP ≤ 1.0
};

struct GPSMeasurement {
    double timestamp;
    Eigen::Vector3d position_enu;  // ENU坐标
    GPSQuality quality;
    double hdop;
    int num_satellites;
    Eigen::Matrix3d covariance;    // 观测协方差
};

3.2 前端里程计模块 — Fast-LIVO2
3.2.1 模块概述
Fast-LIVO2 是紧耦合 LiDAR-Inertial-(Visual) 里程计系统，基于 Error-State Iterated Kalman Filter (ESIKF)。

3.2.2 状态向量


x = [R, p, v, bg, ba, g]^T

其中:
  R  ∈ SO(3)    : 旋转
  p  ∈ R³       : 位置
  v  ∈ R³       : 速度
  bg ∈ R³       : 陀螺仪偏置
  ba ∈ R³       : 加速度计偏置
  g  ∈ R³       : 重力向量
3.2.3 处理流程


IMU数据 ──► 前向传播(预测)
                │
LiDAR帧 ──► 点面残差计算 ──► ESIKF更新
                │
Camera帧 ──► 光度残差计算 ──► ESIKF更新 (可选)
                │
                ▼
          状态后验估计
                │
                ├──► 发布实时位姿
                │
                ├──► 关键帧判断
                │       │
                │       ├── 条件1: 平移 > 1.0m
                │       ├── 条件2: 旋转 > 10°
                │       └── 条件3: 时间 > 2.0s
                │
                └──► 局部地图更新 (ikd-Tree)
3.2.4 关键帧输出格式
C++

struct KeyFrame {

3.3.2 GPS-里程计一致性检验
每收到一个GPS测量:
  1. 计算GPS位置与当前里程计位置的偏差:
     Δp = p_gps - p_odom
  
  2. 马氏距离检验:
     d_M = sqrt(Δp^T × (Σ_gps + Σ_odom)^{-1} × Δp)
  
  3. 判定:
     if d_M > χ²(3, 0.99):  // 自由度3, 99%置信度
       标记为异常, 不使用
     else:
       正常使用, 协方差按质量调整

3.7.3 因子图结构


因子类型:

1. 里程计因子 (Binary Factor)
   连接: KeyFrame_i ─── KeyFrame_{i+1}
   测量: ΔT_odom = T_i^{-1} × T_{i+1}
   协方差: 来自 Fast-LIVO2

2. 回环因子 (Binary Factor)
   连接: KeyFrame_i ─── KeyFrame_j
   测量: ΔT_loop (TEASER++ 输出)
   协方差: 基于内点率和RMSE自适应计算

3. GPS 因子 (Unary Factor)
   连接: KeyFrame_k
   测量: p_gps_enu
   协方差: 基于GPS质量自适应
   
   ┌───────────────────────────────────┐
   │  GPS协方差设定:                    │
   │  EXCELLENT: σ = [0.05, 0.05, 0.1] │
   │  HIGH:      σ = [0.1,  0.1,  0.2] │
   │  MEDIUM:    σ = [1.0,  1.0,  2.0] │
   │  LOW:       σ = [10.0, 10.0, 20.0] │ (几乎不起作用)
   │  INVALID:   不添加因子              │
   └───────────────────────────────────┘

4. 子地图间约束 (Binary Factor, Level 2)
   连接: SubMap_i ─── SubMap_{i+1}
   测量: 子地图锚定帧间相对变换
3.7.4 优化触发策略


触发条件:
  ├── 条件A: 检测到新回环约束 → 触发全局优化
  ├── 条件B: 累积 M 个新子地图 → 触发增量优化
  ├── 条件C: 建图结束 → 触发最终全局优化
  └── 条件D: 手动触发

优化策略:
  • 增量优化: 仅优化新增部分 + 受影响的局部
  • 全局优化: 所有子地图联合优化
  • 最终优化: 多轮迭代直至收敛

6. GPS 处理策略
6.1 GPS 状态机


           ┌───────────┐
     ┌─────│  INIT      │
     │     │ 等待首个   │
     │     │ 有效GPS    │
     │     └─────┬─────┘
     │           │ 收到有效GPS
     │           ▼
     │     ┌───────────┐
     │     │  TRACKING  │◄────────────────────┐
     │     │ GPS正常跟踪│                      │
     │     └─────┬─────┘                      │
     │           │ GPS失效 / 质量下降          │ GPS恢复
     │           ▼                            │
     │     ┌───────────┐                      │
     │     │  DEGRADED  │──────────────────────┘
     │     │ GPS退化    │
     │     │ 纯里程计   │
     │     └─────┬─────┘
     │           │ 长时间无GPS
     │           ▼
     │     ┌───────────┐
     └────►│  LOST      │
           │ GPS完全丢失│
           │ 依赖回环+  │
           │ 里程计     │
           └───────────┘
6.2 GPS 在各阶段的协方差策略


┌────────────┬────────────┬──────────────┬────────────────────────────┐
│  GPS状态    │  协方差策略  │  在前端的作用 │  在后端的作用               │
├────────────┼────────────┼──────────────┼────────────────────────────┤
│ EXCELLENT  │ σ=0.05m    │ 强约束       │ 强一元因子                  │
│ HIGH       │ σ=0.1m     │ 强约束       │ 强一元因子                  │
│ MEDIUM     │ σ=1.0m     │ 弱约束       │ 弱一元因子                  │
│ LOW        │ σ=10m      │ 几乎不用     │ 极弱因子/不添加             │
│ INVALID    │ 不使用      │ 跳过         │ 不添加                     │
│ DEGRADED   │ 渐增σ      │ 逐渐减弱     │ 协方差随时间增大            │
│ LOST       │ 不使用      │ 纯LIO       │ 不添加, 依赖回环           │
└────────────┴────────────┴──────────────┴────────────────────────────┘
6.3 GPS 跳变处理


检测条件:
  |p_gps(t) - p_gps(t-1)| > v_max × Δt + 3σ
  其中 v_max 为最大合理速度

处理:
  1. 标记为异常
  2. 连续3次异常 → 进入 DEGRADED 状态
  3. 连续5次正常 → 恢复 TRACKING 状态

重定位:
  GPS从LOST恢复后:
  1. 等待连续5个一致的GPS测量
  2. 与里程计位置对比, 计算偏移
  3. 如果偏移合理 → 逐步注入GPS因子
  4. 如果偏移异常 → 继续等待
